#THINGSdb

根据自然语言（中文）自动生成系统的工具。


主要开发语言
    
    Java
    
设计背景

    社会分工更加专业化是历史发展的趋势。
    即将到来的技术革命也应是为此服务的，我想。
    
    现在的分工
       专业人士描述业务知识，由IT技术者代为开发自己业务领域的系统工具。
       
    未来的分工
       专业人士自己写关于自己业务领域的系统工具，
       他们是利用自然语言描述的，由专业工具自动生成系统工具。
       这个专业工具，我称之为AI。
       

主要组件:

    1,THINGSdb      负责数据存取;
    2,SDP解释器      负责sdp解析执行;
    3,Stanford NLP  负责自然语言解析（他社）;
    4,J2Cache       二级缓存（他社）;

【THINGSdb】

    是一个key-value数据库
    主要记忆数据与数据关系
    被设计为大数据库

【SDP解释器】

    自定义查询程序=SDP
    将自然语言转成SDP
    执行解释SDP

【Stanford NLP】

    感谢斯坦福大学提供的组件。
    将一句中文进行词法+语法分析。
    针对输出的语法树，进行入库处理
    
【J2Cache】

    感谢J2Cache的组件提供。
    由于THINGSdb的速度太慢，不得不追加缓存机制。

    郁闷的是，速度还是提不上去

    例，web画面，或batch程序
